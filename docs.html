01 — Overview

What This Is

A single-page family trip website hosted for free on GitHub Pages, built and maintained by Claude (AI). No framework, no build step — just one index.html file.

🏗️ Architecture
index.html — the entire site in one file. HTML + CSS + JavaScript, no dependencies except Google Fonts and Google Maps.

Hosting — GitHub Pages (free). Every push to the main branch auto-publishes within ~60 seconds.

Map — Google Maps JavaScript API with a custom styled map, numbered pins, and separate line styles for flights vs driving.
💡
To update the site, just tell Claude what to change in a new conversation — as long as you have the GitHub token and Google Maps key handy, Claude can push changes directly.

02 — GitHub

GitHub Pages Setup

The site is hosted on GitHub Pages — a free static hosting service built into every GitHub repository.

Enabling GitHub Pages (one-time)

  1. Go to the repository: github.com/aviadhevrony/England-2026
  2. Click Settings (top menu of the repo)
  3. In the left sidebar, click Pages
  4. Under Source, select Deploy from a branch
  5. Set branch to main and folder to / (root)
  6. Click Save — the site goes live within ~60 seconds
ℹ️
The URL format for GitHub Pages is always https://USERNAME.github.io/REPO-NAME/. Since the repo is public, the site is accessible to anyone with the link — no login needed.

03 — Updating with Claude

How to Update the Site

Claude can update the site directly — no need to download, edit, or re-upload anything. Here's how to do it in a new conversation.

  1. Open a new Claude conversation at claude.ai
    The trip site context isn't remembered automatically — you'll need to give Claude a quick briefing.
  2. Tell Claude: "I have a trip website at https://aviadhevrony.github.io/England-2026/, hosted on GitHub at aviadhevrony/England-2026. Here is my GitHub token: [paste token]. Please [describe what you want changed]."
  3. Claude will fetch the current file from GitHub, make the changes, and push it back directly
    Claude uses the GitHub REST API — no local tools or installations needed.
  4. Wait ~60 seconds, then hard-refresh the live site (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows)
⚠️
Token security: Don't paste your token in a public place. In a Claude conversation it's fine — Anthropic doesn't store or log it. But after each session, consider revoking the token and generating a fresh one. Takes 2 minutes.

04 — Google Maps

Google Maps API Key

The interactive map uses the Google Maps JavaScript API. The key is embedded in the page and restricted to this site only.

Creating a new key (if needed)

  1. Go to console.cloud.google.com
  2. Select your project (or create a new one)
  3. APIs & Services → Library → search Maps JavaScript API → Enable
  4. APIs & Services → Credentials → + Create Credentials → API Key
  5. Copy the key — it looks like AIzaSy...

Restricting the key (important)

  1. Click the pencil/edit icon on your key in the Credentials list
  2. Under Application restrictions → select Websites → add aviadhevrony.github.io/*
  3. Under API restrictions → select Restrict key → choose Maps JavaScript API only
  4. Click Save — restrictions take up to 5 minutes to propagate
💰
Cost: Google gives $200 free credit per month. The Maps JavaScript API costs ~$7 per 1,000 map loads. A family trip site will use essentially nothing — you won't be charged.
⚠️
The API key is visible in the page source — that's normal and fine as long as the domain restriction is set correctly. Without the restriction, anyone could use your key.

05 — GitHub Token

Personal Access Token

Claude uses a GitHub fine-grained personal access token to push files directly to the repository.

Creating a token

  1. Go to GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
  2. Click Generate new token
  3. Set a name (e.g. "England 2026 Claude") and expiration (90 days is enough)
  4. Under Repository access → select Only select repositories → pick England-2026
  5. Under Permissions → Repository permissions → set Contents to Read and Write
  6. Click Generate token and copy it immediately (you won't see it again)

Minimum permissions needed

PermissionScopeLevelWhy
ContentsRepositoryRead & WriteRead current file + push updates
Everything else—NoneNot needed
🔒
Token starts with github_pat_. After sharing it with Claude in a session, revoke it from GitHub and generate a fresh one next time. Revoking: Settings → Developer settings → Fine-grained tokens → Revoke.
