What we built

The Overview

This is a static single-page website for the Hevrony family England 2026 trip. It was built by Claude (Anthropic) in a conversation, using flight data read directly from Gmail, and deployed to GitHub Pages.

The site lives as a single index.html file in a GitHub repository. No server, no database, no framework — just HTML, CSS, and JavaScript. Claude can update it directly by pushing to GitHub using a personal access token.

💡
The site is rebuilt from scratch each time Claude makes a change — it reads the latest version from GitHub, modifies it, and pushes the updated file back. All history is preserved in GitHub's commit log.

Hosting

GitHub Pages Setup

The site is hosted for free on GitHub Pages. To set it up or re-enable it:

  1. 1
    Go to the repositoryOpen github.com/aviadhevrony/England-2026
  2. 2
    Open SettingsClick the "Settings" tab in the top menu of the repository
  3. 3
    Go to PagesIn the left sidebar, click "Pages" under "Code and automation"
  4. 4
    Set the sourceUnder "Build and deployment" → Source: Deploy from a branch → Branch: main → Folder: / (root) → Save
  5. 5
    Wait ~60 secondsThen visit aviadhevrony.github.io/England-2026/
ℹ️
GitHub Pages is free for public repositories. The site deploys automatically every time a file is pushed — usually within 30–60 seconds.

Claude write access

GitHub Personal Access Token

To allow Claude to push files directly to the repo (without you downloading and uploading), a GitHub personal access token is used. This is a secret key that lets Claude call the GitHub API on your behalf.

Creating a new token

  1. 1
    Go to GitHub token settingsgithub.com/settings/tokens?type=beta
  2. 2
    Click "Generate new token"Choose Fine-grained token (more secure than classic)
  3. 3
    Name itE.g. "Claude – England 2026"
  4. 4
    Set expiration90 days is enough to cover planning through September 2026
  5. 5
    Repository accessSelect "Only select repositories" → pick England-2026
  6. 6
    Set permissionsRepository permissions → Contents → Read and Write. That's the only permission needed.
  7. 7
    Generate and copyToken starts with github_pat_... — paste it to Claude

Token permissions summary

SettingValueWhy
Token typeFine-grainedScoped to one repo only
RepositoryEngland-2026Only this repo, nothing else
Contents permissionRead & WriteNeeded to read and update index.html
All other permissionsNoneNot needed
⚠️
Security note: After each Claude session where you share a token, go to github.com/settings/tokens and revoke it. Generate a fresh one next time. Tokens in chat history should be considered exposed.

Interactive map

Google Maps API Key

The interactive map uses the Google Maps JavaScript API. A free API key is required.

  1. 1
    Go to Google Cloud Consoleconsole.cloud.google.com → create or select a project
  2. 2
    Enable the APIAPIs & Services → Library → search "Maps JavaScript API" → Enable
  3. 3
    Create a keyAPIs & Services → Credentials → + Create Credentials → API Key
  4. 4
    Restrict by websiteEdit the key → Application restrictions → Websites → add aviadhevrony.github.io/*
  5. 5
    Restrict to one APIAPI restrictions → Restrict key → select Maps JavaScript API only → Save
💰
Google gives $200 free credit per month. The Maps JavaScript API costs $7 per 1,000 loads. For a private family trip site with a handful of visitors, you will never be charged — the free tier covers tens of thousands of page loads.
ℹ️
The API key is embedded directly in index.html. It is protected by the website restriction above, so it can only be used on your GitHub Pages domain and is safe to include in the file.

APIs enabled on this key

APIUsed for
Maps JavaScript APIInteractive map, pins, polygons, overlays
Directions APIReal driving routes between stops (click a pin to load)
⚠️
Both APIs must be enabled in Google Cloud Console → APIs & Services → Library and both must be added to the key's API restrictions list, otherwise routes won't load.

Before you travel

UK ETA — Entry Requirement

Israeli passport holders must obtain a UK Electronic Travel Authorisation (ETA) before travelling to the UK. It is checked by the airline at check-in — you cannot board without it.

ℹ️
All 5 travellers need one. Each person applies individually. Cost is £10 per person (£50 total for the group).

How to apply

  1. 1
    Go to the official UK government siteApply at gov.uk/eta or download the UK ETA app (iOS / Android) — search "UK ETA" in the App Store or Google Play.
  2. 2
    Prepare your documentsValid Israeli passport, a photo of your face, email address, and a payment card (£10 per person).
  3. 3
    Complete the applicationTakes about 5–10 minutes per person. Answer a few questions about travel history and purpose of visit.
  4. 4
    Wait for approvalUsually approved within a few hours. You'll receive a confirmation email — no physical document is issued, it's linked to your passport digitally.
  5. 5
    Check the ETA is linked to the right passportMake sure you travel with the exact passport you used to apply.

Key facts

DetailInfo
Cost£10 per person
Validity2 years or until passport expiry (whichever is sooner)
Trips allowedMultiple trips during validity period
Max stay per visit6 months
Processing timeUsually a few hours; can take up to 3 working days
Where it's storedDigitally linked to your passport — no printout needed
Must apply beforeBefore boarding the flight — airlines check at check-in
⚠️
Apply well in advance — don't leave it to the last minute. Although usually fast, processing can take up to 3 working days. Apply at least 2 weeks before departure.
